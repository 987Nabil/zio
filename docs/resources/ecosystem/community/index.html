Community ZIO Libraries

List of first-class ZIO support libraries from the community:

  • caliban — Functional GraphQL backend in Scala
  • cakeless — Wire your cakes automatically into zio environment
  • distage — Staged, transparent and debuggable runtime & compile-time Dependency Injection Framework
  • idealingua — API Definition, Data Modeling and RPC Language, optimized for fast prototyping – like gRPC, but with a human face
  • logstage — Automatic structural logs from Scala string interpolations
  • munit-zio — Lightweight bridge between ZIO and MUnit testing library
  • rezilience — Utilities for resilience and handling of transient errors
  • slf4zio — Simple convenience layer on top of SLF4J for ZIO
  • tranzactio — ZIO wrapper for data access libraries like Doobie or Anorm
  • ZIO Arrow — Haskell Arrow meets ZIO. A deep composition and high performance applications
  • zio-amqp — ZIO Streams based RabbitMQ client
  • zio-aws — Low-level AWS wrapper for ZIO for all AWS services using the AWS Java SDK v2
  • zio-aws-s3 — A lean, simple and efficient ZIO wrapper for AWS Java v2 S3 API by Boris V.Kuznetsov
  • zio-easymock — EasyMock mocking for zio-test
  • zio-email — Purely functional email client
  • zio-event-sourcing — Purely functional concurrent and scalable persistance layer
  • zio-grpc — A native gRPC support for ZIO
  • zio-interop-log4j2 — Fiber aware MDC logging for Log4j 2
  • zio-kinesis — ZIO Streams based AWS Kinesis client
  • zio-magic — Construct ZLayers automagically (w/ helpful compile-time errors)
  • zio-saga — Purely functional transaction management with Saga pattern
  • zio-slf4j — Referentially transparent logging with slf4j
  • zio-slick — Bridge library between ZIO and Slick Functional Relational Mapping Library
  • zio-test-akka-http — Akka-HTTP Route TestKit for zio-test

If you know a useful library that has first-class ZIO support, please consider submitting a pull request to add it to this list.
